# Feature Map

This document summarizes FRIDAY runtime capabilities by subsystem.

## 1. Orchestration Core
- Deterministic plan -> execute -> validate -> report pipeline
- Runtime module boundaries and coordinator flow
- Replayable run history and structured event records

## 2. Policy and Safety
- Risk-tier policy decisions (low/medium/high/critical)
- Approval routing and bounded autonomy controls
- Kill-switch handling and fail-safe execution boundaries

## 3. Security and Trust
- Prompt injection and identity override protection
- Untrusted instruction execution guards
- Social-engineering and policy-anomaly detection
- Incident playbooks and forensic export support

## 4. Memory and Session Intelligence
- Ingestion/index/retrieval with confidence scoring
- Preference-aware memory resolution
- Open-loop register and status-check summaries
- Session carry-over and continuity behavior

## 5. Persona and Communication
- FRIDAY and JARVIS profile behavior
- Addressing preference resolution and mode switching
- Answer-first and no-filler response contracts
- Communication calibration and directive-compliance evaluation
- Compliance drift, correction workflow, and directive audit reporting

## 6. Multimodal and UI Automation
- Screenshot pipeline and OCR/layout analysis
- UI grounding and action planning
- Safety checks for UI action pre/post validation
- Confidence-aware visual fallback behavior

## 7. Control Plane and Physical Layer
- Local/remote connector management and health
- Scoped command templates with rollback support
- Physical connector SDK and simulation-first gating
- Manual takeover, geofence, interlock, and emergency-stop handling

## 8. Reliability and Launch Operations
- SLO and error-budget tracking
- Alert routing and event bus integration
- Backup/restore/disaster recovery workflows
- Release pipeline, canary decisions, and launch readiness review

## 9. Moonshot Capability Program
- Benchmark taxonomy and harness
- Long-horizon and cross-domain evaluation
- Safety regression gating for capability evolution
- Quarterly capability gap reporting and governance review

## 10. Governance Automation (In Progress)
Planned workstream:
- Standardized phase summary/verification/validation artifacts
- Automated milestone audit computation
- Requirements traceability generation
- Governance-aware release gate enforcement
